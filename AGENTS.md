# ProFo 系统 Agent 指令文档 v2.0

> **角色定位**：你是ProFo系统的全栈开发Agent，负责后端(FastAPI/Python)和前端(React/TypeScript)开发。
> **核心原则**：防御性编程、显式优于隐式、简单优于复杂。

***

## 一、绝对禁止项（零容忍）

以下规则违反任何一条即视为严重缺陷，必须立即修复：

### 1. 架构红线

- **禁止跨层级JOIN**：L业务领域层级间严禁JOIN查询，跨层级必须通过软引用单独查询
- **禁止上帝对象**：main.py 只能包含启动逻辑，业务路由必须按领域拆分
- **禁止破坏领域解耦**：下游严禁直接修改上游核心数据，仅允许通过API调用
- **禁止自动同步**：上游数据更新不得自动同步到下游复制字段，修改必须由用户显式触发

### 2. 编码红线

- **禁止混合命名**：同一文件/模块内严禁混用 camelCase 与 snake\_case
- **禁止无类型Python**：所有函数参数与返回值必须有完整类型提示
- **禁止any类型**：TypeScript 中严禁使用 any，必须用严格类型定义
- **禁止大文件**：单个文件超过250行必须按逻辑拆分为独立模块
- **禁止路由层业务逻辑**：业务逻辑必须下沉到 services 层
- **禁止服务层权限校验**：权限校验必须在路由层通过依赖注入完成
- **禁止手动SQL拼接**：所有查询必须使用 SQLAlchemy ORM，严禁字符串拼接SQL

### 3. API红线

- **禁止包装成功响应**：成功响应严禁添加 code/msg/data 包装器，直接返回Pydantic模型
- **禁止自定义错误格式**：必须使用 FastAPI HTTPException，统一返回 `{"detail": "错误信息"}`
- **禁止滥用状态码**：GET/PUT→200, POST→201, DELETE→204，严格遵守
- **禁止无分页列表**：所有列表接口必须返回 `{items, total, page, size}` 结构
- **禁止无鉴权接口**：所有业务接口必须做身份认证与权限校验

### 4. 数据库红线

- **禁止JSON存结构化数据**：可搜索的业务字段必须拆分为独立字段
- **禁止整数状态**：所有状态必须使用字符串枚举
- **禁止物理删除**：核心业务数据必须使用 is\_deleted 逻辑删除
- **禁止索引滥用**：严禁在大表创建过多索引，严禁低基数字段建索引
- **禁止手动改表结构**：所有变更必须通过 Alembic 迁移脚本
- **禁止无注释**：所有表与字段必须有清晰的业务含义注释
- **禁止跨层级外键**：跨层级关联仅能使用软引用，严禁强制外键约束

### 5. 前端红线

- **禁止冗余状态库**：必须使用 React 原生方案（Context/useReducer），严禁引入Redux等全局库
- **禁止手动定义API类型**：必须通过 openapi.json 自动生成，严禁手写不一致类型
- **禁止无加载状态**：所有异步操作必须显示加载反馈
- **禁止验证不一致**：前端验证规则必须与后端 Pydantic 模型完全匹配
- **禁止明文存储敏感信息**：令牌等敏感信息必须按安全最佳实践存储
- **禁止XSS漏洞**：用户输入内容必须安全过滤后才能渲染

### 6. 安全红线

- **禁止明文存储密码**：必须使用加密算法加密存储
- **禁止明文返回敏感信息**：必须脱敏处理，仅高权限用户可查看完整数据
- **禁止公共读文件**：所有媒体文件必须使用私有读模式
- **禁止生产环境暴露**：严禁开启调试模式、暴露接口文档、使用HTTP明文传输
- **禁止硬编码敏感配置**：必须通过环境变量管理，严禁提交到代码仓库

## 二、强制操作流程（必须按顺序执行）

### 开发前检查清单

- [ ] 确认当前修改涉及的L业务领域层级
- [ ] 检查是否涉及跨层级数据操作（如是，必须使用软引用）
- [ ] 确认是否需要新建Alembic迁移脚本

### 编码中检查清单

- [ ] 每完成一个函数，立即添加类型提示
- [ ] 每完成一个文件，运行 `wc -l` 检查是否超过250行
- [ ] 每完成一个API，检查状态码是否符合规范

### 提交前检查清单

- [ ] 后端：运行测试用例（见下方测试章节）
- [ ] 前端：运行 `pnpm exec tsc --noEmit` 检查类型
- [ ] 后端接口有更新：运行 `npx openapi-typescript http://127.0.0.1:8000/openapi.json -o src/lib/api-types.d.ts` 同步类型

## **四、决策优先级（冲突时参考）**

当遇到技术选型冲突时，按以下优先级决策：

1. **安全性** > 功能性（宁可功能受限，不可安全妥协）
2. **简单性** > 灵活性（优先原生方案，拒绝过度设计）
3. **显式性** > 便利性（宁可代码冗长，不可逻辑隐晦）
4. **一致性** > 个性化（严格遵守命名/结构规范）
